# Lane Oil Pattern 3D

볼링 레인 오일 패턴 시트(Kegel / FLEX)의 수치를 그대로 입력하면, 브라우저의 3D 레인 위에
오일 분포를 렌더링하고 내 볼러·볼 스펙으로 샷을 시뮬레이션하는 웹 앱입니다.

> **Live demo**: https://fearnot502.github.io/3DBowlingLane/

## 무엇을 할 수 있나

### 3D 레인 위의 오일 패턴
실제 규격(60ft × 39보드)의 레인에 가이드 도트(6ft), 타깃 화살표(12–16ft), 레인지 파인더(34/40ft),
거터, 우드 핀덱과 10핀까지 재현하고, 그 위에 오일 패턴을 두 가지 모드로 보여줍니다.

- **패턴표 모드** — 인쇄된 시트의 3색 범례를 그대로 재현. 색은 레이어 구성만 표시하고
  (Forward=시안, Reverse=블루, 겹침=네이비), 오일 양은 높이(displacement)로 표현합니다.
- **실제 레인 모드** — PBA 중계 화면처럼 오일이 나무 위의 반투명 광택 글레이즈로 보이는 모드.
  밀도를 부드럽게 압축(Reinhard)해 두꺼운 헤드 구간도 자연스럽게 깊어지고, 보드가 항상 비쳐 보입니다.

레인 폭 배율(기본 5× = 패턴표 그래프 비율, 1× = 실제 비율), Forward/Reverse 레이어 토글,
불투명도·두께 슬라이더, 패턴 상하 반전, 5ft 간격 거리 라벨과 패턴 끝 마커를 제공합니다.

### 샷 시뮬레이션
볼러와 볼 스펙을 입력하면 현재 오일 패턴 위에서 스키드 → 훅 → 롤로 이어지는 실제 샷을
물리 시뮬레이션합니다.

- **볼러 스펙** — 오른손/왼손, 스타일 프리셋(스트로커·트위너·크랭커), 볼 스피드(km/h), 회전수(Rev Rate)
- **릴리스 축** — 축 회전(Axis Rotation, 사이드 로테이션 = 훅 길이) · 축 틸트(Axis Tilt, 스필러 =
  플레어 감소·조기 롤). 기본값 55°/15°에서 값을 바꾸면 훅량과 플레어가 변합니다
- **볼 스펙** — RG(스키드 길이), Differential(전체 훅/플레어), PSA(브레이크포인트 각짐)
- **내 라인** — 스탠스(레이다운) 보드와 타겟(에로우 15ft) 보드를 조절하면 엔트리 보드·진입각·
  브레이크포인트·판정(포켓/두꺼움/얇음/거터)이 실시간으로 표시됩니다. 슬라이더는 실제 장수 읽는
  순서(오른손 40→1)로 배치되고 손에 따라 자동 미러링됩니다
- **라인 추천** — 현재 패턴에서 관용도 높은 윗장(바깥)·아랫장(안쪽) 라인을 탐색해
  근거(진입각, 브레이크포인트, Rule of 31 예상, 미스 관용도, 경로 평균 오일)와 함께 제시합니다

### 3D 재생
궤적을 스키드(시안)·훅(주황)·롤(빨강) 위상별로 색칠해 그리고, 실제 규격의 볼이 회전하며
라인을 따라 굴러갑니다. 재생/일시정지, 속도 조절, 반복, 타임라인 스크러버를 지원하고,
거리별 스피드·슬립·브레이크포인트를 보여주는 궤적 분석 차트가 함께 표시됩니다.

화면 우상단의 **볼 인스펙터 창**을 열면 공만 따로 확대해 볼 수 있습니다. 독립 캔버스지만 메인
재생과 공유 클럭으로 동기화되어, 볼의 회전과 오일 트랙 플레어가 생기는 과정을 그대로 보여줍니다.
드래그로 돌려볼 수 있고, 닫은 뒤 보기 탭에서 다시 켤 수 있습니다.

### 오일 트랜지션
"진행 샷 수"를 올리면 플레이한 라인(볼 트랙)을 따라 헤드·미들 오일이 마르고(브레이크다운)
패턴 끝 뒤로 얇은 필름이 밀려납니다(캐리다운). 실제 레인 모드에서 눈으로 확인할 수 있고,
궤적과 라인 추천에도 반영됩니다(패턴표 모드는 갓 깐 프레시 패턴 유지). 트랙은 처음 적용한
라인으로 고정(스냅샷)되어 스팟을 옮겨도 오일 표시는 그대로이고 볼 반응만 바뀌며,
초기화 버튼으로 프레시 상태로 되돌립니다.

### 내 셋업 (아스날)
현재 볼러·볼 스펙과 라인을 이름 붙여 저장해 두고, 어떤 패턴에서든 불러와 비교할 수 있습니다.

### 패턴 입력
- **표 붙여넣기** — `START STOP LOADS SPEED BUFFER [TANK] CROSSED START END FEET T.OIL`
  형식의 행을 그대로 붙여넣으면 파싱 (TANK 열·행 번호 유무 자동 인식)
- **PDF 가져오기** — 텍스트형 시트는 메타데이터와 표를 자동 추출, 이미지형 시트는 1페이지를 미리보기로 렌더링
- **AI 가져오기** — 이미지형 시트를 AI 챗으로 변환한 텍스트를 붙여넣는 플로우
- 내장 샘플: **2026 JINSEUNG A TYPE** (45ft, 29.5 mL)

가져온 패턴과 셋업은 브라우저(`localStorage`)에 저장되어 새로고침 후에도 유지됩니다.

### 분석
보드별 오일 분포 스택 차트(시트 하단 차트와 동일 형식), 피크 보드, 실측 패턴 길이,
거리별 오일 단면(3D 레인에 위치선 표시), 시트의 Track Zone Ratio 표를 제공합니다
(Track Zone은 머신 고유 알고리즘 값이므로 재계산하지 않고 표시만 합니다).

## 화면 구성

- **데스크톱** — 좌측 사이드바가 **패턴 / 보기 / 플레이 / 분석** 4개 탭으로 나뉩니다
- **모바일** — 3D가 항상 배경에 보이고, 아래에서 끌어올리는 바텀 시트(접힘/절반/전체 3단 스냅)에 컨트롤이 담깁니다
- 자주 쓰는 컨트롤(오일 모드·레이어·라벨·핀·폭 배율)은 3D 화면 위 플로팅 툴바에 노출됩니다
- 라이트 / 다크 테마 토글 — 3D 배경·바닥까지 테마 연동, 선택은 저장됩니다

## 내부 동작 원리

### 보드 표기 해석
Kegel 표기는 **각 거터에서 안쪽으로 센 번호**입니다:

| 표기 | 의미 | 절대 보드 |
|------|------|-----------|
| `4L` | 왼쪽 거터에서 4번째 | 4 |
| `4R` | 오른쪽 거터에서 4번째 | 36 (= 40 − 4) |
| `20` | 센터 보드 | 20 |

따라서 `4L-4R`은 보드 4~36의 **넓은** 스팬, `14L-15R`은 보드 14~25의 **좁은** 스팬입니다.
시트의 `CROSSED = LOADS × 지나간 보드 수`로 검증됩니다.

샷 시뮬레이션의 **볼러 보드**는 이와 별개로, 사용 손 쪽 거터에서 안쪽으로 센 번호입니다
(오른손 1-3 포켓 / 왼손 1-2 포켓 = 17.5보드). 좌표를 볼러 보드로 계산해 한 코드 경로로
좌우손을 모두 처리합니다.

### 오일 밀도 모델 (`lib/oilModel.js`)
각 패스는 보드 범위 × 거리 범위가 만드는 사각형에 오일을 도포한다고 보고,
해상도와 무관한 밀도(µl per board·foot)로 그리드에 누적합니다:

```
density = T.OIL ÷ (보드 수 × 피트 길이)
```

- `T.OIL`이 없으면 `LOADS × 50µl × 보드 수`로 추정 (LOADS=0 이동 행은 오일 없음)
- 버퍼 브러시의 전폭 얇은 필름(Forward: 파울라인→버프아웃, Reverse: 브러시 드롭→파울라인)과
  마지막 로드를 파울라인까지 끌고 가는 드래그를 합성해 인쇄 그래프와 같은 형태를 재현
- 좌우로 가우시안 블러(≈2.2보드)를 적용해 브러시가 옆으로 번지는 어깨 그라데이션 표현
- 합산 오일 총량은 시트의 Forward/Reverse/Volume Total과 일치 (JINSEUNG A: 19.05 / 10.45 / 29.5 mL)

### 볼 물리 모델 (`lib/ballMotion.js`)
0.25ft씩 레인을 내려가며 매 스텝의 **로컬 오일 밀도로 마찰**을 정하는 격자 기반 시뮬레이션입니다.
마찰이 실제 레인처럼 세 가지 일을 합니다:

1. 볼을 **스핀업**(slip 1 → 0): 스키드 → 훅 → 롤 전환
2. 슬립이 남아 있는 동안 볼을 **옆으로 밀어냄**(훅)
3. 전진 **속도를 깎음**

볼 스펙은 볼러가 쓰는 감각 그대로 매핑됩니다 — **RG**↑ 스핀업 저항(늦은 브레이크),
**Diff**는 플레어 포텐셜(총 훅량 스케일), **PSA**는 브레이크포인트에서의 급격함
(부드러운 아크 ↔ 날카로운 플립), **축 회전**↑ 훅 길이 증가, **축 틸트**↑ 플레어·훅 감소.
라인 추천도 같은 시뮬레이션을 여러 라인에 반복 실행해 미스 관용도를 평가하는 방식입니다.

### 회전 & 트랙 플레어 (`components/BallPath.jsx`)
현실의 회전 역학을 최대한 반영했습니다:

- **렙업(rev-up)** — 스키드 구간은 릴리스 회전수로, 롤 구간은 노면 속도(`v/r`)로 도는 블렌딩
- **축 이행** — 릴리스 때 진행방향으로 기운 스핀축(사이드 로테이션 = 훅)에서 슬립이 줄며
  순수 엔드오버엔드(롤)로 이행
- **트랙 플레어(보타이)** — 오일 링을 **회전수 기준**으로 남겨(렙수↑ = 링 촘촘), 스핀축이
  한 평면 안에서 이행하도록 해 모든 링이 두 매듭점을 통과하는 실제 오일트랙 **보타이** 형태를
  형성합니다. 이행량은 Diff에 비례, 축 틸트에 반비례하며 슬립에 비례해 누적되므로,
  롤에 들면(slip→0) 이행이 멈춰 링들이 매듭으로 수렴합니다
- 링 데이터는 샷마다 **결정론적으로 미리 계산**해, 메인 볼·인스펙터·타임라인 스크럽이
  정확히 같은 플레어를 공유합니다

### 오일 트랜지션 모델 (`lib/oilModel.js` — `ageOilGrid`)
진행 샷 수를 받아 **트랙을 따라** 패턴을 노화시킵니다 — 헤드/미들에 브레이크다운(오일 제거),
패턴 끝 뒤에 캐리다운(얇은 필름 도포). 트랙은 프레시 그리드에서 뽑은 샷 경로의 스냅샷을 사용해
순환 의존을 차단합니다. 물리(시뮬레이션·라인 추천)에는 항상 반영되고, 화면 표시에는
실제 레인 모드에서만 반영됩니다.

## 참고

볼 물리와 오일 트랜지션은 휴리스틱 모델입니다. 실제 레인 표면·볼 상태·환경에 따라 결과가
달라질 수 있으니, 정성적인 경향을 읽는 도구로 사용하세요.
